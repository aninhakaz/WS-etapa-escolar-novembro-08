# WS-etapa-escolar-novembro-08

Nesses 3 módulos a produção de um aplicativo chamado SisCall foi realizado, com telas como: Login, Cadastro, Home, Abertura de chamado, Perfil e Chat. No primeiro módulo foi realizado a criação e modelagem das telas de acordo com o front-end, reproduzindo e comparando fielmente o layout, as cores, a tipografia, etc. Mantendo os alinhamentos, os tamanhos aproximados, tudo o máximo possível para estar fiel ao apêndice fornecido. No segundo módulo, as ações foram criadas a partir do desenvolvimento dentro do back-end, disponível no Swagger e as criações das API calls dentro do flutter flow. Nessa etapa, foi essencial para que o aplicativo ser funcional, permitindo ações dentro do aplicativo, como por exemplo: Entrar no chat de conversa, cadastrar um usuário, dentre outras ações. O módulo de número 3, tinha como critério e objetivo analisar e aprimorar tudo para que o aplicativo funcione. Foi utilizado funções como actions (ações), como navigate, backend, etc. Tudo está documentado nos Modulos A,B,C do docs, inclusiva as telas. Para avaliar se está tudo correto, é nescessario usar uma funçao do próprio Flutterflow, que vai rodar e mostrar exatamente como seria no aplicativo mobile.
